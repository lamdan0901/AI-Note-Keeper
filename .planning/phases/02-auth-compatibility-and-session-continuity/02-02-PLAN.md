## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-05 | S | upgrade-session endpoint | mitigate | Require authenticated continuity checks on user existence and issue short-lived access tokens with strict issuer/audience claims. |
| T-02-06 | R | refresh/logout lifecycle | mitigate | Record refresh token IDs and revocation timestamps for replay detection and auditable token-state transitions. |
| T-02-07 | I | auth route responses | mitigate | Keep all failures on standardized AppError envelope and avoid disclosing whether username or password caused login failure. |
| T-02-08 | D | auth endpoints | mitigate | Apply schema validation and endpoint-level throttling hooks for login/refresh/upgrade to limit brute-force and replay pressure. |

</threat_model>
