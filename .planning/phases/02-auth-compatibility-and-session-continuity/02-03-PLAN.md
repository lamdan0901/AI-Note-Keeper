<threat_model>

## Trust Boundaries

| Boundary                | Description                                                                                                        |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------ |
| web/mobile->api         | Client auth context state transitions depend on remote auth responses and token/session transport semantics.       |
| client-storage->runtime | Persisted session state (localStorage/SecureStore) is trusted input at bootstrap and must be validated before use. |

## STRIDE Threat Register

| Threat ID | Category | Component                   | Disposition | Mitigation Plan                                                                                                                    |
| --------- | -------- | --------------------------- | ----------- | ---------------------------------------------------------------------------------------------------------------------------------- |
| T-02-09   | S        | client bootstrap upgrade    | mitigate    | Validate legacy session shape before upgrade-session call and reject malformed userId payloads early.                              |
| T-02-10   | I        | web session storage         | mitigate    | Keep refresh/session secrets in httpOnly cookies only and avoid writing token secrets to localStorage/sessionStorage.              |
| T-02-11   | T        | mobile secure storage state | mitigate    | Persist token/session data only through SecureStore helpers with strict schema parsing and clear-on-parse-failure behavior.        |
| T-02-12   | D        | auth context refresh loops  | accept      | Temporary retry/backoff behavior is accepted in this phase; monitor for repeated 401 loops and cap retries in follow-up if needed. |

</threat_model>
