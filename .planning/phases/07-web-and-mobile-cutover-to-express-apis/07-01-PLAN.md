<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser session state to Express auth endpoints | Untrusted client requests may use stale/forged access tokens. |
| Access token refresh path to protected resource requests | Incorrect retry logic can leak auth failures or create replay loops. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | S | Web API bearer token transport | mitigate | Use access-token resolver from auth state and retry only after validated refresh response. |
| T-07-02 | T | 401 refresh-and-retry flow | mitigate | Enforce single retry guard and fail closed on second 401. |
| T-07-03 | I | Web session persistence | mitigate | Keep refresh token cookie-only and do not persist refresh token in localStorage. |
| T-07-04 | D | Auth loop risk | mitigate | Add transport regression tests for retry-once and refresh failure branches. |
</threat_model>
