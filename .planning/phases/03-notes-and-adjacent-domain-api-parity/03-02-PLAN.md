## Trust Boundaries

| Boundary                  | Description                                                                        |
| ------------------------- | ---------------------------------------------------------------------------------- |
| client->subscriptions API | Untrusted lifecycle and scheduling inputs enter billing/reminder derivation logic. |
| client->device-token API  | Untrusted token/device payloads can target idempotent upsert/delete paths.         |
| domain service->postgres  | Ownership and dedupe correctness depend on SQL predicates and unique constraints.  |

| Threat ID | Category | Component                    | Disposition | Mitigation Plan                                                                                          |
| --------- | -------- | ---------------------------- | ----------- | -------------------------------------------------------------------------------------------------------- |

| T-03-06   | S        | device token ownership       | mitigate    | Require authenticated user context and scope all token mutations by `(user_id, device_id)` checks.       |
| T-03-07   | R        | token mutation auditing      | mitigate    | Keep deterministic upsert/delete responses and tests proving idempotent behavior for replayed requests.  |
| T-03-08   | I        | notification ledger boundary | mitigate    | Add explicit regression test blocking backend persistence/API exposure for `notification_ledger` (D-11). |

</threat_model>
