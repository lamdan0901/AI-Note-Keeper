<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| client->AI endpoints | Untrusted transcripts and clarification payloads enter parsing and provider flows. |
| AI service->external provider | Third-party response content and availability can be malformed or unavailable. |
| AI endpoints->rate limiter | Request burst patterns can degrade service reliability without throttling. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | I | AI provider output | mitigate | Normalize provider output through strict DTO pipeline and never return raw provider payloads. |
| T-03-10 | D | AI endpoints | mitigate | Enforce endpoint-level per-user throttling with explicit rate-limit error contract (AICP-03). |
| T-03-11 | T | fallback logic | mitigate | Use deterministic local fallback for missing/failing provider and keep transcript normalization deterministic. |
| T-03-12 | S | AI route access | mitigate | Require verified access token context before invoking parse/clarify service paths. |
</threat_model>
