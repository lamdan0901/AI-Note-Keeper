<threat_model>

## Trust Boundaries

| Boundary                   | Description                                                      |
| -------------------------- | ---------------------------------------------------------------- |
| client->notes API          | Untrusted sync payloads cross into ownership and mutation logic. |
| notes service->postgres    | LWW and dedupe decisions persist authoritative state.            |
| bearer token->user context | Auth claims become ownership scope for all note mutations.       |

## STRIDE Threat Register

| Threat ID | Category | Component                      | Disposition | Mitigation Plan                                                                                          |
| --------- | -------- | ------------------------------ | ----------- | -------------------------------------------------------------------------------------------------------- |
| T-03-01   | S        | `requireAccessUser` middleware | mitigate    | Verify access JWT signature, issuer, audience, and expiry before extracting user identity.               |
| T-03-02   | T        | `syncNotes` mutation path      | mitigate    | Enforce strict ownership predicate and LWW timestamp comparison; reject stale and cross-user writes.     |
| T-03-03   | R        | note change events             | mitigate    | Persist dedupe keys (`note_id`, `user_id`, `operation`, `payload_hash`) for replay traceability.         |
| T-03-04   | D        | sync endpoint                  | mitigate    | Validate request schema limits batch shape and size; short-circuit malformed payloads at route boundary. |

</threat_model>
