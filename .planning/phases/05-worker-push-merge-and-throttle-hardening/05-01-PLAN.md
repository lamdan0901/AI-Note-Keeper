<threat_model>

## Trust Boundaries

| Boundary                          | Description                                                                                        |
| --------------------------------- | -------------------------------------------------------------------------------------------------- |
| worker cron trigger -> dispatcher | Periodic trigger can replay or overlap during restarts and must not produce duplicate side effects |
| dispatcher -> queue enqueue       | Enqueue operations can partially fail and must not advance watermark prematurely                   |
| queue payload -> handler          | Untrusted payload replay can attempt duplicate occurrence execution                                |

## STRIDE Threat Register

| Threat ID | Category | Component                     | Disposition | Mitigation Plan                                                                                               |
| --------- | -------- | ----------------------------- | ----------- | ------------------------------------------------------------------------------------------------------------- |
| T-05-01   | T        | dispatch watermark update     | mitigate    | Update cron_state only after enqueue batch success per D-02; failed enqueue path keeps prior watermark        |
| T-05-02   | D        | due-reminder scanner window   | mitigate    | Enforce MAX_LOOKBACK_MS guard and bounded `[since, now]` scan for every run (JOBS-01)                         |
| T-05-03   | R        | occurrence execution identity | mitigate    | Derive stable `noteId-triggerTime` event identity and use unique enqueue keys for idempotency (D-03, JOBS-03) |
| T-05-04   | E        | worker delivery semantics     | mitigate    | Use at-least-once delivery with idempotent handlers, not at-most-once shortcuts (D-04)                        |

</threat_model>
